Line search and step size settings

The step size dictates how far one traverses along a local descent direction. More specifically, the step size $gamma_t$ is used at each iteration to determine how much the next iterate moves towards the new vertex:

\[x_{t+1} = x_t - \gamma_t (x_t - v_t).$$\]

$\gamma_t = 1$ implies that the next iterate is exactly the vertex, a zero $\gamma_t$ implies that the iterate is not moving.

The following are step size selection rules for Frank Wolfe algorithms. Some methodologies (e.g. FixedStep and Agnostic) depend only on the iteration number and induce series $\gamma_t$ that are independent of the problem data, while others (e.g. GoldenSearch and Adaptive) change according to local information about the function; the adaptive methods often require extra function and/or gradient computations. The typical options for convex optimization are Agnostic or Adaptive.

All step size computation strategies are subtypes of LineSearchMethod.

Line search and step size methods

FrankWolfe.Adaptive — Type

Slight modification of Adaptive Step Size strategy from this paper.

source
FrankWolfe.Agnostic — Type

Computes step size: 2/(2 + t) at iteration t.

source
FrankWolfe.Backtracking — Type

Computes step size via the Backtracking Line Search method.

source
FrankWolfe.Emphasis — Type

Emphasis given to the algorithm for memory-saving or not. The memory-saving mode may not be faster than the default blas mode for small dimensions.

source
FrankWolfe.FixedStep — Type

Constant step size given by gamma0

source
FrankWolfe.Goldenratio — Type

Computes a step size via Golden Section Search.

source
FrankWolfe.LineSearchMethod — Type

Line search method to apply once the direction is computed.

source
FrankWolfe.Nonconvex — Type

Computes a step size for nonconvex functions: 1/sqrt(t + 1).

source
FrankWolfe.RationalShortstep — Type

Computes a 'Rational Short step' step size: sum((x - v) .* gradient ) // (L * sum((x - v) .^ 2)), where L is the Lipschitz constant of the gradient, x is the current iterate, and v is the current Frank-Wolfe vertex.

source
FrankWolfe.Shortstep — Type

Computes the 'Short step' step size: dual_gap / (L * norm(x - v)^2), where L is the Lipschitz constant of the gradient, x is the current iterate, and v is the current Frank-Wolfe vertex.

source
